Basic Metric Configuration Help

The Basic Metric collects rudimentary run-time performance metrics for any society to which it is added. The results can then be viewed in the Performance Analyzer.

Throughout task production, metrics are collected for each agent. Once the agent is started, it will start producing metrics at Start Delay + current time. The agent will then continue to report its metrics at every interval Sample Interval until the requested number of metrics samples Maximum Number of Samples are collected.

Metrics contain some standard statistics that are always collected. These include:

Sample Length in Seconds

This statistic is the CPU seconds used by the Node during a sample.
Note: This CPU statistic requires the separate CpuClock shared library, included with the scalability module. See the Scalability module's install instructions for installing the CpuClock library. If you do not install this library, you will see a warning exception, which can be ignored.

Memory used by Node in Megabytes

This statistic indicates the total memory allocated to the Node in megabytes.

Option Metrics and Parameters

The remaining parameters allow you to customize metrics collection and which metrics are collected.

Max Number of Metric Samples

 The maximum number of metrics samples to collect.

Metric Sample Interval

 Specifies the interval, in seconds, that metric samples are collected. The first metric is collected at current time + Start Delay and subsequent metrics are collected at every interval until the maximum number of samples are collected or the society stops running.

Start Delay

 Start delay specifies a delay, in seconds, before metrics collection start.

Task Verb

 The Task Verb parameter allows you to request that the plugin count the tasks with the given Verb that go by.
It will also count the time since the first task with that Verb went by.
For example, use this to count the Transport tasks, or to see how long after a Determine Requirements Task things get busy.

Setting the task verb adds three new statistics:
  • Time since first Task with the Verb
  • Number of Tasks in this interval with the verb
  • Total Tasks seen with the verb
Note: See know issues for more information on Task Verb The following parameters indicate various services that contain statistics. Collection of these statistics can be turned on/off by setting the value of the parameter. Default is: off for all services

Blackboard Statistics Service

 Turning on Blackboard statistics adds:
  • Count of Assets on this Agent's Blackboard
  • Plan Elements on this Blackboard
  • Tasks on the Blackboard
  • Total Objects on this Blackboard
