Servlet Group Insertion Recipe Configuration Help

The Servlet Group Insertion Recipe allows users to select groups of individual servlets from the list of available servlets below. These servlets can be added to individual agents as appropriate or to all agents based on the query selected by the user. The target query comes from the file 'PopulateDb.q' or the user created file 'recipeQueries.q'.

Component Fields

Name Type Range
Servlet String True, False
Target Component Selection Query String
_Number of New Servlets Integer 0 - MAX
Classname of Servlet String
Path of Servlet String


Parameters

Servlet

 Specifies whether the particular corresponding servlet will be loaded. If the values is True, the agent will contain the following:
org.cougaar.core.servlet.SimpleServletComponent('servlet class name','path')
The above item will NOT be included if the value is False
Note: Users requiring a servlet loader other than SimpleServletComponent should use Specific Insertion Recipe to add their servlet.

Target Agent Selection Query

 Specifies a defined query in a .q file. This query is used to selected the exact components that should be modified with this recipe. Base, sample queries are in CIP/configs/common/PopulateDb.q
Users should create their own queries in a file called "recipeQueries.q", which should be placed somplace on the Cougaar Config Path: in the directory from which you run CSMART, or in CIP/configs/common, for example.
IMPORTANT: All recipes query names must begin with "recipeQuery"!

_Number of New Servlets

 The number of new servlets that you would like to add that are not included in the list presented. If you set this number to greater than 0 (and hit return to set your selection), then a number of new fields appear, one pair, appropriately numbered, for each new servlet you add:

Classname of Servlet

 The full classname of the servlet that you wish to add. For example for the PlanViewServlet, the classname is as follows:
org.cougaar.planning.servlet.PlanViewServlet

Path of Servlet

 The path that the browser uses to find the particular servlet. Generally the path begins with a '/'. For example for the PlanViewServlet, the path is as follows:
/tasks
