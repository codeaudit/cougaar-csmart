Component Collection Recipe

Documentation in progress.....

With this recipe, you may build up an arbitrary collection of Agents and Plugins to add to a society. Agents will be available for assignment to Nodes as normal. Other Components will be inserted in the Agents or NodeAgents you specify using the target query. Each component can have its own Target Query, by selecting the "Override Parent Target" menu option. Target queries are SQL queries defined in CIP/configs/common/PopulateDb.q for the built-in queries, or in the user-created file "recipeQueries.q" which should go in a similar location. Note that Agents created with this recipe will not have any Oplan related information in the database, if that is important.

To add an Agent, use the pop-up menu on the root folder.

To remove an Agent, use the pop-up menu on the Agent folder.

To add a Plugin, or Other Component, use the pop-up menu on the root folder or the Agent folder, depending on where you want to add it. The drop-down lists of components are editable, and are populated from the list of Plugins / Binders currently known in the database.

To edit a parameter in your Asset, edit the values in the right pane. Within an Asset, you may add Relationships or PropertyGroups. For each case, to add another one, use the corresponding pop-up menu.

To add a Parameter to or remove a Plugin, Binder, or Other Component, use the pop-up menu on that component. When adding parameters to these components, parameters are automatically named so that they will be added at runtime in the order in which you add them here, named "param-0", "param-1", etc.

When adding parameters to PropertyGroups, you will be asked for a parameter name. Note that the names of parameters of Property Groups include the type of Object used to fill in that value. For example, OrganizationPG has one Parameter, whose name is:
Roles (Role)
and whose value is a list, something like "TransportationProvider, StrategicTransportationProvider". This means that the slot in the PropertyGroup is called "Roles", and the Java Class of each object is "Role". In this case, there are many values, so the slot contains a Collection.
The "INI" equivalent lines would be something like:

[OrganizationPG]
Roles                    Collection<Role>       "TransportationProvider, StrategicTransportationProvider"

Spelling errors, referring to a non-existent field or Property Group, or other errors will not be noticed until runtime.

To remove a parameter, use the pop-up menu on that parameter. Note that the remaining parameters will continue to be added according to the numbers given in their names.

Times should be specified in a standard Java parsable format, eg: MM/DD/YY HH:MI AM.

Elements in a collection should be separated by a comma and a space.

Components within agents now have a Priority field. The priority indicates the load priority of the component in it's container relative to other children components loaded at the same time at the same containment point. The default component priority is: COMPONENT
