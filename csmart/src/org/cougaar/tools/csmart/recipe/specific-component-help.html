Specific Component Recipe

Specific Component Recipes allow users to insert a specific component in any society at the set of places indicated by a database query. For example, use this recipe to add a Plugin to an Agent, or to a Node (ie, to that Node's Node Agent, by using a target query that selects Nodes).
A Specific Component Recipe contains the following parameters:

Component Fields

Name Type Range
Class Name String
Component Name String
Number of Arguments Integer 0..MAX
Priority String HIGH, INTERNAL, BINDER, COMPONENT, LOW, STANDARD
Target Component Selection Query String
Type of Insertion String One of: "plugin", "node binder", or "agent binder"


Plus zero or more of the following, depending on the Number of Arguments:

Value X String


Specific components can be added to a society, such as a plugin in an agent, or a new binder. Creating a Specific Component is an easy way to quickly modify your society configuration.

Class Name

 Class Name of the Component.
Example: org.cougaar.tools.csmart.SamplePlugin

Component Name

 Complete Name of the Component.
Usually, this is the same value as Class Name. If you want to insert the same plugin in the same agent twice, pick a unique name.

Number of Arguments

 Specifies the number of arguments required for this component. Modification of this value creates new parameters used to specify the values of each argument.

Priority

 Specifies the component load priority in it's container relative to other child components loaded at the same time at the same containment point.

Target Component Selection Query

 Specifies a defined query in a .q file. This query is used to selected the exact components that should be modified with this recipe. Base, sample queries are in CIP/configs/common/PopulateDb.q
Users should create their own queries in a file called "recipeQueries.q", which should be placed someplace on the Cougaar Config Path: in the directory from which you run CSMART, or in CIP/configs/common, for example.
IMPORTANT: All recipes query names must begin with "recipeQuery"!

Type Of Insertion

 Specifies the type of component: "plugin", "agent binder", or "node binder".

Value X

 There will be one value parameter for every Argument added with the Number of Arguments parameter. This value specifies the value of the argument required for the component.
