Agent Insertion Recipe Configuration Help

The Agent Insertion Recipe allows users to create an arbitrary agent for inclusion in a society, along with a skeletal Organization Asset. In order to add plugins to the agent, use one of the other Recipes, and be sure that the other recipe is listed after this recipe in the Experiment Builder.

Note: As of COUGAAR v9.0, PlanServerPlugin is not included in agents created with this recipe. Note: As of COUGAAR v9.4, OrgAsset is automatically included for all Agents. This is to address bug #937. The inclusion of an OrgAsset Adds the following plugins to the agent:
  • org.cougaar.mlm.plugin.organization.OrgDataPlugin
  • org.cougaar.mlm.plugin.organization.OrgReportPlugin
  • ClusterPG

Parameters

Agent Names

 The list (comma separate) of the agent names (all identical) which you wish to create. NOTE: Currently, only one name is permitted.

Alternate Type Identification

 An alternate identification for the type of this Organization.

Nomenclature

 A verbose nomenclature for this Organization.

Class Name

 The class for this Agent. The default is almost always correct.

Number of Relationships

 The number of relationships which this Organization should have with other organizations. If you set this number to greater than 0 (and hit return to set your selection), then a number of new fields appear, one pair, appropriately numbered, for each relationship you add:

Relationship

 The name of the agent with which to have the relationship - note that the available values will appear in a drop-down box, showing all the agents known in the database. Note that this means you may only create relationships with Agents already added to other Experiments.

Role

 The role which this agent should have, relative to the other agent specified above.

Type Identification

 The type identification for this agent.
