Impact Configuration Help

Impact adds a Kinetic or Cyber Event to the Society.

Component Fields

Name Type Range
Cyber Events Integer 0 - MAX
Kinetic Events Integer 0 - MAX
Samples Per Second String 0 - 4
In Messages Per Second String 0 - ?
Out Messages Per Second String 0 - ?


Currently, there are two types of impacts, Cyber and Kinetic impacts. A typical configuration can have multiple impacts of each type. The number of impacts in a configuration is specified by 'Cyber Events' and 'Kinetic Events'.

Cyber Event

 A Cyber Event simulates a cyber event such as a denial of service attack on a specific target.

Kinetic Event

 A Kinetic Event simulates a real world event such as a flood on a specific target.

Samples Per Second

 Specifies how many times per second Messages should be released from the queues. For example, if this is set to "0.25", then the Messages will only be released from the queues every 4 seconds and would result in "bursts" of I/O every 4 seconds.

A good value is "2", which will make the Binder's behavior mostly transparent to the application. The maximum supported value is "4". Note: This value should not be changed from the default value.

In Messages Per Second

 Specifies how many messages can be received by an Agent per second. This is used to specify an upper-bound on the Agent's simulated network performance.
For example, "2" would allow at most two messages per second to enter the Agent, with additional messages buffered up.

Out Messages Per Second

 Specifies how many messages can be send by an Agent per second.
