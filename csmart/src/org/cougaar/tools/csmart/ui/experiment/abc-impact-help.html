Impact Configuration Help

The Impact adds an Kinetic or Cyber Impact to the Society.
  • Cyber Events [Integer, 0 - MAX]
  • Kinetic Events [Integer, 0 - MAX]
  • Samples Per Second [String, 0 - 4]
  • In Messages Per Second [String, 0 - ?]
  • Out Messages Per Second [String, 0 - ?]
Currently, there are two types of impacts, Cyber and Kinetic impacts. A typical configuration can have multiple impacts of each type. The amount of impacts in a configuration is specified by 'Cyber Events' and 'Kinetic Events'.

A Kinetic Impact simulates a real world event such as a flood on a specific target.

'Samples Per Second' specifies how many times per second Messages should be released from the queues. For example, if this is set to "0.25", then the Messages will only be released from the queues every 4 seconds and would result in "bursts" of I/O every 4 seconds.

A good value is "2", which will make the Binder's behavior mostly transparent to the application. The maximum supported value is "4". This value should not be changed from the default value.

'In Messages Per Second' specifies how many messages can be received by an Agent per second. This is used to specify an upper-bound on the Agent's simulated network performance.

For example, "2" would allow at most two messages per second to enter the Agent, with additional messages buffered up.

'Out Messages Per Second' specifies how many messages can be send by an Agent per second.
