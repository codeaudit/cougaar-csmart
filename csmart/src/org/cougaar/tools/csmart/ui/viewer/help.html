Cougaar Society Management, Analysis, and Reporting Tool


CSMART is a loosely integrated collection of tools for configuratin and running Cougaar societies, and understanding the results. For complete details, see the CSMART User's Guide on the Cougaar web site (http://www.cougaar.org). CSMART is used to create a society configuration, store it in a configuration database, and then run adn control that society. For details on installing CSMART, see the User's Guide, or the included Installation guide (csmart/doc/InstallAndTest.html).

CSMART Tool Launcher

The CSMART Tool Launcher is the primary interface to CSMART, and integrates the various CSMART tools. From this interface, users create societies, experiments, and recipes. They then use the various tools to configure, run, monitor, and analyze Cougaar societies. The CSMART tools are displayed across the top of this window. These tools are generally used in left-to-right order. The tools and their functions are:
  • Configuration Builder: Edit society or recipe properties, such as the number of agents, or parameters for a given Plugin.
  • Experiment Builder: Put together the pieces of an Experiment, by adding one or more Recipes to the basic society (including Impacts or Metrics). This is the tool where you assign agents to nodes, and nodes to hosts, and set run-time command line arguments for the nodes.
    For database experiments, this tool allows the user to add Recipes (Impacts and Metrics and others) to experiments, and to map agents to nodes, and nodes to hosts. Multiple trials are not supported in database experiments. In general however, you use this tool for constructing multiple trials, varying parameters of your society, or to understand some behavior in more detail. To run a Society, you must include it in an Experiment.
  • Experiment Controller: Run, stop or abort the experiment, and display output from the experiment. Output from the Nodes can be filtered or searched, and CSMART can notify you of certain Strings in the output. This is often referred to as the "CSMART Console", for it is the interface for running large Cougaar societies.
  • Society Monitor: Examine a running society (or saved data from a previous run), by looking at the contents of its Blackboard. Display graphs of agents and plan objects, and some simple metrics. This tool may be run stand-alone for use with non-CSMART Cougaar societies. See the User's Guide.
  • Performance Analyzer: Display metric results from an experiment in progress, or a completed experiment.

Be sure to separately run the Node Server on each host where you want to run a Node, using one of the supplied Server startup scripts.
To quit CSMART, select File->Exit from the main window, and kill each of the Servers.

Configuring an Experiment

Currently, CSMART allows building and running two types of experiments:
  • experiments based on more elaborate Cougaar societies in which the basic society definition is obtained from a database, referred to as "database" experiments
  • experiments based on very regular societies, most suitable for scalability and other such testing, referred to as "built-in" experiments (temporarily disabled)

To run an experiment from the database:

  1. Click the right mouse button on the "Default Workspace" to display a pop-up menu, or use the File menu. Select "New Experiment -> From Database" to display a list of experiments defined in the database. Select the experiment from the database and provide a name for the experiment.
  2. Select the threads and groups for the experiment from the dialog displayed. You must select at least one thread (or class of supply). When you exit the dialog, any modified information is written to the database.
  3. Select the Experiment Builder and assign agents to nodes and nodes to hosts. When you exit this tool, any modified information is written to the database.
  4. Select the Experiment Controller and select the "Run" button near the top of its window.

To run a tool:

  • Select the desired experiment, recipe or society (possibly creating a new society or experiment),
  • Select a function (edit, run) on the menu on the selected experiment or society in the Workspace, or
  • Select the desired function from the File menu, or
  • Select the appropriate toolbar button.
To activate a tool: The CSMART tools are activated as follows. Note that if a folder is selected, then all items in the folder are selected. Hence, multiple experiments and societies can be selected at once.
  • Configuration Builder: Always enabled. When the Configuration Builder tool is selected, a separate Configuration Builder is launched for each selected society or recipe. If no recipe is selected, a new recipe is created. In some situations, the user is prompted whether they want to edit, copy, or view the recipe for which they opened the Configuration Builder. In view mode, the component is not modifiable, but the user may view the current configuration.
  • Experiment Builder: Always enabled. When the Experiment Builder tool is selected, a separate Experiment Builder is launched for each selected experiment. In some situations, the user is prompted to edit, copy, or view the selected Experiment. In View mode, the experiment is not modifiable, but the user may view the current configuration.
  • Experiment Controller: Activated if an experiment that contains a society is selected. Only one Experiment Controller tool is allowed.
  • Society Monitor: Activated if an experiment is running. To view previously saved graphs, or to monitor a non-CSMART Cougaar society, use this tool in its stand-alone form. See the User's Guide for details.
  • Performance Analyzer: Activated if an experiment is selected.
Be sure to select File -> Save before exiting each tool to ensure your work is saved to the database. In addition, CSMART automatically saves your workspace as you go, in the current working directory. To retrieve previously saved work, Select File-> Open Workspace from the main CSMART window. Look for the file named according to your workspace name.

Workspace

The large lower panel displays the Workspace. The Workspace is used to create, edit, rename or delete folders, experiments, societies, and recipes. This is how you organize your work. Note that your work is automatically saved in the current working directory, under the name you give your workspace. To open a previously saved workspace, use the File menu option. You may organize items in the workspace however your prefer.

  • Select an item: Point to the item and click the left mouse button. Selecting a folder selects all the items in the folder.
  • Create, edit, rename, or delete: Point to the item and click the right mouse button; select the appropriate action from the pop-up menu displayed. You may access the same functions from the File menu in the Launcher window. You may also rename components by clicking on the name directly.
  • Arrange items in workspace: Point to the item, hold the left mouse button and drag the item to a new location.
Once you have created an initial society or experiment, use the appropriate tools to fine tune your component. Then run your experiment using the Experiment Controller.

Societies

For CSMART purposes, a Society is a collection of Cougaar Agents, grouped into one or more Communities. Societies are constructed from a template, and then parametrized to fully specify the desired Communities, Agents, and Plugins. Currently, two templates are supported for "built-in" societies. Use the Configuration Builder to tailor a society. For database experiments, the society is specified as part of the experiment and cannot be reconfigured with the Configuration Builder.

Experiments

To run a society, it must be included in an Experiment. An Experiment is, in general, made up of one or more trials. Trials are constructed by varying some parameter - a Plugin parameter, which Agents are run, what Recipes are included, the distribution of Agents to Nodes or Nodes to Hosts, etc. For each trial, there may be some results - statistics about the run that are collected by one of the specified metrics (see below). Experiments created from a database cannot vary any parameter, and thus have only one trial.

Recipes

A Recipe is a set of rules for modifying a society, by adding one or more components to the society. Recipes are meant to be used for multiple societies. There are several built-in recipes. By tailoring these and saving them, users can then retrieve recipes from the database for later reuse. For example, one recipe might add a certain plugin to every agent in the society. This Recipe could then be used for any society.
To create a recipe, select "New Recipe" from the File menu or "Default Workspace" pop-up menu. Then select "Built In" or "From Database". You will get a list of the available recipes of each type. Once you have loaded the recipe, you may configure it by selecting "Configure" from the pop-up menu on the recipe or from the File menu. Be sure to save the recipe when done.
To add a recipe to an Experiment, drag it onto the "Recipes" folder in the Experiment Builder. Recipes can also be duplicated, deleted, or renamed by using the pop-up menu or File menu options.

Metrics

A metric is a kind of recipe, a set of run-time statistics about the running society, which you wish to collect. CSMART allows you to add one or more to your experiment. Examples might include average and peak CPU utilization, total number of tasks in the log plan, etc.
Currently, both built-in societies always include a fixed set of metrics. Database societies do not, and you may add one. The frequency of sampling however is controlled from within the Society Configuration tool. See the User's guide for further details.

Impacts

An impact is a kind of recipe, an external effect you wish to impose on a running Cougaar society. For example, loading the network with background traffic, killing nodes, telling the system some inventory has been lost, etc. These impacts might be real (really loading up the network), or simulated (via a Binder on the MessageTransport or some other mechanism).

Notes:

  • If an appropriate item (experiment or society) is not selected in the Workspace, then the toolbar button will not be activated.
  • Changes to your workspace are saved automatically.
